# 通用数据采集与分析方法论

## 一、专利导航分析总框架

### 1.1 核心逻辑

专利导航分析遵循“**数据采集 → 数据处理 → 专利导航分析 → 成果产出**”的基本流程。

### 1.2 五大分析模型

| 报告类型 | 核心分析模型 | 方法论依据 |
|----------|--------------|------------|
| 产业规划类 | 方向—定位—路径模型 | GB/T 39551.3 |
| 区域规划类 | 竞争力—匹配度—质量模型 | GB/T 39551.2 |
| 企业经营类 | 目标评估—风险分析—决策建议模型 | GB/T 39551.4 |
| 研发活动类 | 立项评价—研发辅助模型 | GB/T 39551.5 |
| 人才管理类 | 遴选—评价模型 | GB/T 39551.6 |

## 二、信息采集方法论

### 2.1 专利数据库选择

#### 推荐数据库（按优先级排序）

| 优先级 | 数据库 | 说明 |
|--------|--------|------|
| 第一优先 | 智慧芽 (Patsnap) | 覆盖中国及全球专利数据，内置专利价值评分和机器学习工具 |
| 第一优先 | Derwent World Patents Index (DWPI) | 人工分类增强的黄金标准 |
| 第二优先 | INPADOC | 欧洲专利族数据覆盖能力强 |
| 免费替代 | USPTO、EPO Espacenet、WIPO PATENTSCOPE、CNIPA | 数据量有限，但可满足基础检索需求 |

#### 数据源要求
- 目标数据库需覆盖1亿+全球专利，以支撑全面分析
- 必须支持专利族去重
- 必须提供结构化IPC/CPC分类字段
- 必须提供申请人、发明人、法律状态和法律事件数据

### 2.2 信息分类

| 信息类型 | 具体内容 | 来源渠道 |
|----------|----------|----------|
| 专利信息 | 专利文献（著录项目、权利要求、说明书） | 专利数据库（CNABS、DWPI、SIPOABS等） |
| 产业信息 | 产业规模、结构、趋势、产业链 | 国家统计局、工信部、行业协会、券商研报 |
| 政策信息 | 国家/地方规划、产业政策、指导意见 | 政府官网、政策数据库 |
| 企业信息 | 发展历程、营收、市场占有率、研发投入 | 企业年报、Wind、企查查 |
| 科技信息 | 研发投入、研发人员、论文 | 科技统计年鉴、中国知网、Web of Science |
| 人才信息 | 发明人、技术团队 | 专利数据库、人才申报材料 |

### 2.3 技术分解方法

#### 2.3.1 技术分解的三种方法

| 方法 | 适用场景 | 注意事项 |
|------|----------|----------|
| 专利分类法 | 技术边界清晰、专利分类体系完善的领域 | 需验证分类与产业的对应关系 |
| 行业分类法 | 传统成熟产业 | 需咨询产业专家 |
| 学科分类法 | 基础研究领域 | 不适用于新兴产业 |

#### 2.3.2 技术分解步骤

```
构建技术分解表 → 产业专家调研验证 → 基于检索结果调整 → 确定最终分解表
```

#### 2.3.3 技术分解表示例

| 一级分支 | 二级分支 | 三级分支 | 关键词/分类号 | 技术边界说明 |
|----------|----------|----------|---------------|--------------|
| 上游 | A1 | A1a | IPC B01D | [描述] |
| 中游 | B1 | B1a | 关键词：XX | [描述] |
| 下游 | C1 | C1a | IPC G01N | [描述] |

### 2.4 检索策略构建

#### 2.4.1 双轴混合检索法

专利导航检索采用**双轴策略**：
- **关键词**：跨越分类边界捕捉技术概念
- **IPC/CPC分类号**：捕捉关键词遗漏的精确技术类别

#### 2.4.2 检索式结构

```
检索式 = (关键词1 OR 关键词2 OR ...) AND (分类号1 OR 分类号2 OR ...) AND (申请人/发明人)
```

#### 2.4.3 关键词构建步骤

| 步骤 | 操作 | 示例（碳纤维） |
|------|------|----------------|
| **第一步** | 确定核心技术的中英文名称、同义词、缩写 | “碳纤维”、“carbon fiber”、“CF” |
| **第二步** | 包含上游原材料、下游应用、工艺术语、替代路线 | “聚丙烯腈”、“PAN”、“预氧化丝”、“原丝” |
| **第三步** | 识别并排除不相关含义 | 排除非相关医疗场景 |
| **第四步** | 布尔组合 | (碳纤维 OR CF) AND (PAN OR 聚丙烯腈) |

#### 2.4.4 IPC分类映射

| 步骤 | 操作 |
|------|------|
| **第一步** | 确定核心IPC分组（参考国家标准） |
| **第二步** | 对照已知关键专利交叉检查，包含子分组 |
| **第三步** | 与关键词组合：(关键词组合) OR (IPC分类号组合) |

#### 2.4.5 分类号选择

- IPC分类号（国际通用）
- CPC分类号（更细粒度）
- 其他：EC、FI/FT

## 三、数据处理方法论

### 3.1 数据处理步骤

```
数据去重去噪 → 数据项规范化 → 数据标引
```

### 3.2 专利族去重

**原因**：同一发明在多个国家申请会产生多个专利文件，直接计数会夸大指标。

**方法**：
- 按家族ID对专利分组
- 统计唯一家族数，而非单个文件数
- 地理分析时，按国家成员过滤家族

**输出**：去重后的唯一专利族数据集

### 3.3 数据去重去噪

| 噪声类型 | 处理方法 |
|----------|----------|
| 同族专利重复 | 按优先权去重 |
| 不同检索式重复 | 合并去重 |
| 实用新型与发明重复 | 保留发明 |
| 不相关主题 | 人工筛选剔除 |

### 3.4 实体归一化

**企业实体标准化**：
- 将子公司名称统一到母公司名下
- 处理名称变更（合并、收购、品牌重塑）
- 建立实体映射表
- 示例：FANUC CORPORATION统一为“发那科”

**发明人姓名标准化**：
- 处理中文姓名变体（同一人不同字形）
- 处理音译变体（中→英姓名形式）
- 按“发明人+机构”分组提高准确性

### 3.5 数据项规范化

| 字段 | 规范内容 |
|------|----------|
| 申请人名称 | 母公司统一 |
| 日期格式 | 统一为YYYY-MM-DD（中文格式“2023年01月15日” → “2023-01-15”） |
| 国别/地区 | 统一为标准代码（CN、US、JP等） |
| 技术分支 | 按技术分解表标引 |

### 3.6 数据标引

#### 3.6.1 标引类型

| 标引类型 | 说明 | 示例 |
|----------|------|------|
| 技术分支标引 | 按技术分解表分类 | 上游/中游/下游 |
| 技术功效标引 | 解决的技术问题和达到的效果 | 提高效率/降低成本 |
| 申请人类型标引 | 企业/高校/科研单位/个人 | 企业 |
| 地域标引 | 来源国/目标市场 | 中国/美国 |

#### 3.6.2 标引流程

```
确定标引维度 → 制定标引规则 → 人工/自动标引 → 抽检校验
```

### 3.7 质量检查

- 确认无重复家族残留
- 检查关键字段缺失/空值
- 验证IPC代码格式一致性
- 交叉检查申请人名称一致性

## 四、专利导航分析方法论

### 4.1 专利态势分析

#### 4.1.1 分析维度

| 维度 | 分析内容 | 图表类型 |
|------|----------|----------|
| 全球态势 | 申请趋势、技术产出格局、技术生命周期 | 折线图、气泡图 |
| 国内态势 | 国内申请趋势、技术构成、区域分布 | 折线图、饼图 |
| 区域态势 | 区域专利分布、法律状态、维持年限 | 柱状图、堆积图 |
| 企业态势 | 企业专利布局、技术分支分布 | 矩阵图、气泡图 |
| 竞争态势 | 主要申请人排名、专利布局对比 | 条形图、雷达图 |

#### 4.1.2 数量趋势分析（多阶段叙述结构）

| 阶段 | 特征 | 叙述模板 |
|------|------|----------|
| 萌芽期 | 早期零散申请，技术探索阶段 | “专利申请量在XX年以前处于萌芽期，年均申请量不足XX件。” |
| 加速期/发展期 | 稳步增长，商业可行性得到验证 | “XX-XX年进入加速期，主要得益于XX。” |
| 爆发期 | 快速增长，市场扩展 | “XX年后进入爆发期，XX申请量迅速增长，成为全球增长的主要驱动力。” |

**分析步骤**：
1. 绘制年度申请量折线图
2. 识别拐点（显著增减节点）
3. 为各阶段标注原因说明（政策、市场、技术突破）
4. 在同一图表上比较多个地理层级

**叙述示例**：
> “从全球专利申请趋势来看，相关领域专利申请量呈现明显的三阶段增长特征：萌芽期（1960-1989）年均申请量不足50件，以基础研究和技术探索为主；发展期（1990-2009）年均申请量增长至200件左右，随着量产技术的成熟，加工技术逐步实现工业化；爆发期（2010至今）年均申请量突破1000件，在轻量化需求驱动下，专利申请呈现指数级增长态势。”

#### 4.1.3 技术生命周期判断（S曲线）

**目的**：判断技术处于成长期、成熟期还是衰退期。

**方法**：
- X轴：独立申请人数量（创新广度）
- Y轴：专利申请量（创新强度）
- 按时间绘制轨迹

| 阶段 | 特征 |
|------|------|
| 导入期/萌芽期 | 低申请量、低申请人数量 |
| 成长期 | 申请量快速上升、申请人数量增加 |
| 成熟期 | 申请量见顶、申请人数量趋于稳定 |
| 衰退期 | 申请量下降、申请人数量缩减 |

**应用**：
- 对每个技术子领域应用S曲线分析
- 利用S曲线阶段边界定义主题建模的时间窗口
- 识别新兴与成熟技术领域

### 4.2 地理漏斗分析

**四级递进分析（必须执行）**：

| 层级 | 分析重点 |
|------|----------|
| 全球 (Global) | 建立全球竞争格局 |
| 国家 (National) | 聚焦目标国家（通常为中国） |
| 省份 (Provincial) | 缩小至省份或重点区域 |
| 城市 (City) | 聚焦报告服务的目标城市 |

**每一层级需分析**：
- 专利数量与趋势
- TOP-20申请人排名
- IPC分类分布
- 专利类型比例
- 法律状态分布

**嵌套分析模式**：
> “全球XX产业呈现XX趋势。聚焦中国，XX产业专利申请量占全球XX%。聚焦XX省，该产业专利占比为XX%。进一步分析XX市重点企业，发现XX问题。”

### 4.3 申请人/创新主体分析

#### 4.3.1 TOP-N排名分析

- 按专利族总数排名
- 按类型分类：企业、高校、科研院所、个人
- 计算集中度指标（CR3、CR5、CR10）
- 跨地理层级比较

#### 4.3.2 企业/创新主体画像维度

| 分析维度 | 指标 | 解读 |
|----------|------|------|
| 技术实力 | 专利申请量、授权量、有效量 | 数量反映规模 |
| 技术质量 | 核心专利数量、被引次数 | 质量反映影响力 |
| 技术布局 | 技术分支分布、地域分布 | 布局反映战略 |
| 技术动向 | 近年申请趋势、新进入领域 | 动向反映方向 |

#### 4.3.3 重点企业画像需包含

- 历史专利趋势（折线图）
- 技术主题分布（IPC柱状图）
- 专利类型构成
- 法律状态分布
- 核心发明人团队
- 协作网络（共同申请人图）

### 4.4 技术分析

#### 4.4.1 技术路线图绘制方法

1. 梳理技术发展关键节点
2. 识别核心专利
3. 分析技术演进路径
4. 预判未来方向

#### 4.4.2 技术功效矩阵

- **横轴**：技术手段/技术分支
- **纵轴**：达到的技术效果
- **气泡大小**：专利数量

### 4.5 专利质量评估

#### 4.5.1 多维度质量指标

| 维度 | 指标 | 解释 |
|------|------|------|
| 法律状态 | 有效/失效/审中比例 | 有效比例越高 = 保护越强 |
| 专利类型 | 发明与实用新型比例 | 发明比例越高 = 创新含量越高 |
| 同族规模 | 申请国家数量 | 同族越大 = 商业价值越高 |
| 被引次数 | 前向引用次数 | 引用越多 = 技术影响力越大 |
| 专利价值评分 | 数据库ML评分 | 经济价值综合指标 |
| 法律事件 | 异议、无效、转让 | 活动频次反映战略重要性 |

#### 4.5.2 专利价值金字塔

- 按价值区间对专利分层
- 按地理层级绘制堆叠柱状图
- 比较各区域的价值集中度

### 4.6 IPC分类分析

**热点识别**：
- 按IPC分组统计专利族数量（TOP-10）
- 跨地理层级比较分布
- 识别集中与分散模式

**差距分析**：
- 识别全球数量高但本地数量低的IPC分组
- 标记为技术差距或发展机会
- 典型模式：“两极强化，中游塌陷”

**IPC密度热力图**：
- 行：IPC分组
- 列：时间段
- 颜色强度：专利数量
- 可视化技术焦点的演变

## 五、各类型专利导航专属方法

### 5.1 产业规划类：方向—定位—路径模型

```
产业发展方向导航 → 区域产业发展定位 → 区域产业发展路径导航
```

### 5.2 区域规划类：竞争力—匹配度—质量模型

```
创新竞争力指数 × 创新匹配度指数 = 创新质量评价指数
```

### 5.3 企业经营类：目标评估—风险分析—决策建议

| 子类型 | 核心分析 | 关键步骤 |
|--------|----------|----------|
| 投资并购对象遴选 | 技术创新能力评估 | 专利分析→企业信息关联→对象确定 |
| 投资并购对象评估 | 专利稳定性+侵权风险 | 专利核查→稳定性评价→风险评估 |
| 企业上市准备 | 专利资产+侵权风险 | 专利核查→风险排查→应对策略 |
| 技术合作开发 | 合作主题+对象选择 | 技术分析→申请人筛选→背景调查 |
| 技术引进 | 技术主题+持有人 | 技术分析→专利筛选→持有人调查 |
| 产品开发 | 产品方向+研发路径 | 技术分解→竞争对手分析→专利布局 |

### 5.4 研发活动类：立项评价—研发辅助

```
立项评价：环境评价→技术态势→技术壁垒→立项风险
研发辅助：技术优化→风险规避→专利布局
```

### 5.5 人才管理类：遴选—评价

```
人才遴选：专利筛选→发明人分析→人才名单
人才评价：真实性→匹配性→创新能力→使用风险
```

## 六、政策分析框架

### 6.1 政策时间线构建

**步骤**：
1. 收集相关政策文件（国家和地方层面）
2. 按类型分类：战略规划、资金支持、税收优惠、标准制定、人才引进
3. 创建时间线表格：文件名称、发布机构、日期、核心内容
4. 映射政策强度（每年政策数量）

### 6.2 跨国比较

**比较维度**：
- 政策演进时间线：各国何时开始布局？
- 政策工具组合：资金支持 vs. 法规监管 vs. 标准制定
- 政策强度：政策文件数量与频率
- 政策效果：与专利产出和产业发展的相关性

### 6.3 企业案例研究

**每个代表企业需分析**：
- 企业发展历程与创立背景
- 获得的关键政策支持
- 技术发展轨迹
- 专利策略演进
- 对目标地区的启示

## 七、质量控制要点

| 环节 | 质量控制要点 |
|------|--------------|
| 信息采集 | 查全率≥85%、查准率≥85%、来源可靠 |
| 数据处理 | 去重去噪准确、格式规范、标引准确 |
| 专利导航分析 | 模型有效、方法恰当、结论可靠 |
| 成果产出 | 系统性、科学性、规范性、可操作性 |

## 八、报告生成与资产管理（适用于自动化生成）

### 8.1 内容规模控制

| 报告类型 | 目标字数 |
|----------|----------|
| 综合型专利导航报告 | 约250,000中文字符（±10%） |
| 聚焦型报告 | 150,000 – 200,000字 |
| 单章目标 | 15,000 – 20,000字 |

**内容要求**：必须具备深度分析 + 数据支撑 + 结构完整性，禁止空洞描述或重复扩写。

### 8.2 输出格式规范

| 项目 | 规范 |
|------|------|
| 最终交付格式 | Markdown文件 + Word文件 |
| 文档结构 | 摘要 → 正文（各章节）→ 参考文献 |
| 图表嵌入语法 | `![图X-X 标题](/mnt/user-data/outputs/assets/charts/fig_X_X_name.png)` |
| 图表存储路径 | `/mnt/user-data/outputs/assets/charts/` |
| 数据文件存储路径 | `/mnt/user-data/outputs/assets/data/`（CSV或JSON格式） |

### 8.3 参考文献规范

**正文引用**：必须使用可点击Markdown链接

```
[citation:来源名称](URL)
```
