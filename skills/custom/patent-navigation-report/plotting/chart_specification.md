# 专利导航报告——图表与可视化规范

本参考资料提供了专利导航报告中所有图表类型的规范，包括每种图表的使用场景、数据要求及格式标准。

---

## 1. 图表类型目录

### 1.1 折线图

**使用场景**：
- 专利申请量随时间变化趋势
- 技术生命周期发展轨迹
- 多区域趋势比较

**数据要求**：
- X轴：时间（年份或月份）
- Y轴：数量或百分比
- 多条数据系列进行比较

**格式标准**：
- 线条粗细：主要系列2-3px，次要系列1.5px
- 数据标记：仅在关键拐点处显示数据点
- 颜色：各系列使用不同颜色，并配图例
- 标注：直接在图表上添加阶段标签（萌芽期、加速期、爆发期）

---

### 1.2 柱状图

**使用场景**：
- TOP-N申请人排名
- IPC分类号分布
- 区域专利量比较

**数据要求**：
- 分类轴（申请人名称、IPC分类号、区域）
- 数值（专利数量、百分比）

**格式标准**：
- 默认按降序排列
- 在柱上显示数值标签
- 相关图表间使用一致的配色方案
- 为便于阅读，限制显示TOP-10或TOP-20

---

### 1.3 横向柱状图

**使用场景**：
- 申请人名称较长时（提高文本可读性）
- 带有详细标签的比较排名

**格式标准**：
- 从上到下按降序排列
- 必要时截断长标签并加省略号
- 在柱状图末端显示数值标签

---

### 1.4 饼图

**使用场景**：
- 专利类型分布（发明/实用新型/外观设计）
- 申请人类型分布（企业/高校/科研院所）
- 法律状态分布（有效/失效/审查中）

**数据要求**：
- 整体中的组成部分（各部分之和必须为100%）
- 最多限制为5-7个扇区

**格式标准**：
- 显示百分比标签
- 突出显示重要扇区
- 跨报告使用一致的配色映射

---

### 1.5 堆叠柱状图

**使用场景**：
- 时间窗口内的主题占比演变
- 跨区域的多类别比较
- 按区域划分的专利价值等级分布

**数据要求**：
- 每组包含多个类别
- 类别之间应互斥

**格式标准**：
- 每个类别使用不同的颜色
- 包含图例
- 空间允许时在柱上显示总值

---

### 1.6 S型曲线图

**使用场景**：
- 技术生命周期分析
- 创新广度与创新强度映射

**数据要求**：
- X轴：独特申请人数量（创新广度）
- Y轴：专利申请量（创新强度）
- 带有阶段标签的时间序列轨迹

**格式标准**：
- 绘制带有时间标注的轨迹线
- 标记阶段边界（导入期、成长期、成熟期、衰退期）
- 使用箭头指示轨迹方向
- 直接在图表上添加阶段标签

---

### 1.7 热力图

**使用场景**：
- IPC密度矩阵（IPC分组 × 时间段）
- 区域技术聚焦比较
- 技术关联性分析

**数据要求**：
- 两个分类维度
- 数值强度值

**格式标准**：
- 使用顺序颜色标度（浅色→深色）
- 在单元格内包含数值标签
- 添加行合计和列合计

---

### 1.8 雷达图

**使用场景**：
- 多维度的创新能力比较
- 区域竞争力评估

**数据要求**：
- 最多5-8个维度
- 归一化数值（0-100标度）

**格式标准**：
- 使用半透明填充区域
- 按20%间隔显示网格线
- 清晰标注每个维度

---

### 1.9 散点图

**使用场景**：
- 专利数量与质量相关性分析
- 技术集中度与多元化分析
- 创新投入与产出分析

**数据要求**：
- 两个连续变量
- 可选气泡大小表示第三维度

**格式标准**：
- 标注关键数据点
- 若存在相关性，添加趋势线
- 使用颜色编码区分类别

---

### 1.10 网络图

**使用场景**：
- 共同申请人合作网络
- 引文网络
- 技术共现关系

**数据要求**：
- 节点（实体）和边（关系）
- 边权重表示关系强度

**格式标准**：
- 节点大小与中心度成正比
- 边粗细与关系权重成正比
- 按类别对节点进行颜色编码
- 使用力导向布局

---

### 1.11 流向图

**使用场景**：
- 区域间的技术转移
- 专利转让流向
- 跨境许可模式

**数据要求**：
- 源实体和目标实体
- 流量值

**格式标准**：
- 使用带比例的曲线箭头
- 按类别进行颜色编码
- 包含流量值标签

---

### 1.12 统计表

**使用场景**：
- 带有多个指标的TOP-N排名
- 政策文件时间线
- 企业概况摘要

**格式标准**：
- 为便于阅读，行颜色交替显示
- 标题行加粗
- 数字右对齐，文本左对齐
- 表格下方标注来源

---

## 2. 图表编号体系

### 规范
- 图：图X-X（Figure X-X，其中第一个X为章节号）
- 表：表X-X（Table X-X）
- 每章内按顺序编号

### 示例
- 图7-1：全球碳纤维专利申请量趋势
- 表7-2：全球TOP-20专利申请人排名

## 3. 配色方案标准

### 主色调
| 颜色 | 色值 | 使用场景 |
|------|------|----------|
| 蓝色 | #1E3A8A | 主要数据系列、全球层面 |
| 红色 | #DC2626 | 中国数据系列、突出显示 |
| 绿色 | #16A34A | 增长指标、正面 |
| 橙色 | #EA580C | 警示、差距指标 |
| 紫色 | #7C3AED | 次要数据系列 |
| 灰色 | #6B7280 | 第三级数据、基线 |

### 地理区域配色
| 区域 | 颜色 |
|------|------|
| 全球 | #1E3A8A |
| 中国 | #DC2626 |
| 日本 | #16A34A |
| 美国 | #EA580C |
| 欧洲 | #7C3AED |
| 其他 | #6B7280 |

## 4. 图表标注标准

### 强制性标注
- 带有清晰描述名称的图表标题
- 带单位的坐标轴标签
- 数据来源标注（例如："数据来源：智慧芽专利数据库"）
- 数据截止日期说明

### 可选标注
- 趋势图上的阶段标签
- 关键拐点标记
- 因果解释标注
- 比较基准

---

## 5. 图表位置指南

### 章节内结构
1. 图表应紧跟在首次提及该图段落后放置
2. 图表标题位于图表下方（而非上方）
3. 每个图表后附简要解读文字

### 正文引用格式
- "如图7-1所示，全球碳纤维专利申请量呈现明显的三阶段增长特征。"
- "由图7-1可见，2010年是中国专利申请量的转折点。"
- "这主要得益于国家政策的持续支持和产业资本的加速进入。"

## 6. 图表资产保存规范

### 6.1 图表文件保存路径

生成图表时同步保存两类文件：

**图表图片文件：`outputs/assets/charts/`**
```
outputs/assets/charts/
├── fig_1_1_title.png
├── fig_1_2_title.png
├── ...
├── fig_5_1_title.png
├── table_2_1_title.png
└── ...
```

**绘图数据文件：`outputs/assets/data/`**
```
outputs/assets/data/
├── data_1_1_title.csv          ← CSV格式（表格数据）
├── data_1_1_title.json         ← JSON格式（结构化数据）
├── data_5_1_title.csv
├── data_5_1_title.json
└── ...
```

### 6.2 文件命名规范

- 图表图片：`fig_X_X_brief_title.png` 或 `table_X_X_brief_title.png`
- 数据文件：`data_X_X_brief_title.csv` + `data_X_X_brief_title.json`
- 使用英文小写和下划线
- X_X 为图号/表号（如 fig_1_1、table_2_3）

### 6.3 正文引用格式

```markdown
![图5-1 全球碳纤维专利申请趋势（1990-2024）](/mnt/user-data/outputs/assets/charts/fig_5_1_trend.png)

*图5-1 全球碳纤维专利申请趋势。数据来源：智慧芽专利数据库，数据截止2024年12月*
[citation:Patsnap Database](https://www.zhihuiya.com/)
```

### 6.4 数据文件内容要求

CSV文件应包含生成图表所用的完整数据：
```csv
year,global,china,japan,usa
1990,120,0,80,40
1991,135,5,82,42
...
```

JSON文件应包含结构化的数据和元信息：
```json
{
  "chart_id": "fig_5_1",
  "title": "全球碳纤维专利申请趋势",
  "chart_type": "line",
  "data_source": "智慧芽专利数据库",
  "data_cutoff": "2024-12-31",
  "x_axis": "年份",
  "y_axis": "专利申请量（件）",
  "series": ["全球", "中国", "日本", "美国"],
  "data": [...]
}
```

- [ ] 图表具有描述性标题

- [ ] 图表附有解读文字
- [ ] 颜色符合标准

- [ ] 图表大小足以清晰阅读
- [ ] 无多余的装饰性元素
- [ ] 图号遵循编号规范
- [ ] 图表图片已保存到 `/mnt/user-data/outputs/assets/charts/`
- [ ] 绘图数据已保存到 `/mnt/user-data/outputs/assets/data/`
- [ ] 正文中使用 `/mnt/user-data/outputs/assets/charts/` 绝对路径引用图片
