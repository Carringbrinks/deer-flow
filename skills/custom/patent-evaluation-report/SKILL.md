# 专利检索评价报告生成技能

## 报告标准结构

报告严格遵循以下结构（对应参考文档中的表格行）：

| 章节 | 内容 | 参考文档行 |
|------|------|-----------|
| 表头元数据 | 申请号/申请日/优先权日/发明名称/申请人/专利类型/检索依据 | Row 0-5 |
| A | 检索使用的数据库 | Row 6 |
| B | 检索使用的分类号（IPC 第8版） | Row 7 |
| C | 检索使用的中文与外文关键词 | Row 8 |
| D | 相关专利文件（含类型、公开号、公开日期、标题、涉及权利要求编号） | Row 9-14 |
| E | 相关非专利文件（含类型、期刊名、文章标题、相关部分、涉及权利要求编号） | Row 15-19 |
| F | 评述和结论 | Row 20-21 |
| 尾部 | 检索人员和检索单位 + 完成日期 | Row 22 |

**F 评述和结论内部结构**:
1. **新颖性评述**: 逐项对比技术方案与对比文件，判断是否符合专利法第22条第2款
2. **创造性评述**: 分析区别技术特征、技术问题和有益效果，判断是否符合专利法第22条第3款
3. **实用性评述**: 判断是否能够在产业上制造或使用，符合专利法第22条第4款
4. **检索结论**: 融合五维度综合评价（技术、法律、市场、经济、战略），参考浙江省专利奖评价指标

## 核心工作流

### 阶段一：专利文件解析

1. **读取上传文件**
   - 支持格式：`.docx`, `.pdf`, `.txt`, `.xml`（专利全文）
   - 使用 `python-docx` 解析 `.docx` 文件
   - 提取核心元数据：
     - 申请号、公开号、申请日、公开日
     - 申请人、发明人、代理机构
     - IPC/CPC 分类号
     - 专利标题、摘要、关键词
     - 权利要求书（独立权利要求 + 从属权利要求）
     - 说明书（技术领域、背景技术、发明内容、具体实施方式）
     - 引用参考文献/对比文件

2. **结构化信息抽取**
   - 核心技术方案与创新点
   - 技术效果与优势
   - 保护范围分析（权利要求层级结构）

### 阶段二：背景技术检索与分析

#### 2.1 专利数据库检索（概念性）

**数据源**: 壹专利 (Yi Patent)、智慧芽 (Zhihuiya)

- 检索策略：基于 IPC 分类号 + 关键词组合检索
- 目标：获取同族专利、引用专利、被引专利、对比文件
- 分析维度：
  - 技术演进路线与时间线
  - 核心对比文件的技术方案对比
  - 专利申请趋势（年度申请量变化）
  - 申请人/权利人分布

#### 2.2 学术文献检索（强制使用 MCP-ClickHouse）

**数据库**: CERS.ClickHouse
- **CNKI 表**: `CERS.CNKI`（约 1677 万条记录）
- **WOS 表**: `CERS.WOS`（约 1695 万条记录）

**CNKI 表结构**:
```
GUID, journal, journal_key, year, issue, ISSN, title, abstract, keyword,
subject, creator, institute, fund, DOI, date, cited_num, page, pageCounts,
isCore, url, download_count
```

**WOS 表结构**:
```
guid, PT, AU, TI, SO, DT, AB, DE, ID, PU, TC, Z9, CR, PY, VL, IS_,
J9, JI, WC, SC, UT, OA, HC
```

**检索策略**:
```sql
-- CNKI 检索示例：基于关键词和主题分类
SELECT title, abstract, keyword, creator, institute, year, cited_num, journal, isCore
FROM CERS.CNKI
WHERE (title LIKE '%关键词%' OR abstract LIKE '%关键词%' OR keyword LIKE '%关键词%')
  AND subject LIKE '%IPC分类%'
ORDER BY cited_num DESC
LIMIT 20;

-- WOS 检索示例：基于主题词和高被引筛选
SELECT TI, AB, AU, SO, TC, PY, UT
FROM CERS.WOS
WHERE (TI LIKE '%keyword%' OR AB LIKE '%keyword%')
ORDER BY TC DESC
LIMIT 20;
```

**分析产出**:
- 技术领域研究热点与前沿方向
- 高被引学术论文的核心发现
- 产学研结合度分析（高校/研究机构 vs 企业）
- 技术成熟度评估（基于文献时间分布与引用趋势）

#### 2.3 行业与市场信息检索（概念性）

**数据源**: 互联网公开信息、行业报告

- 目标：获取行业规模、增长率、竞争格局、政策环境
- 分析维度：
  - 行业生命周期阶段（导入期/成长期/成熟期/衰退期）
  - 主流技术路线与市场份额
  - 核心竞争企业及其产能/成本/市占率
  - 国家政策与行业标准

### 阶段三：评述与结论撰写

#### F.1 新颖性评述

**写作规范**（严格参照参考文档格式）:

```
一、新颖性
   对比文件公开日均早于本专利，属于现有技术。
   
   [逐项描述对比文件公开的技术方案]
   对比文件1公开了...
   对比文件2公开了...
   对比文件3公开了...
   
   本专利涉及...
   
   [分析区别技术特征]
   在XXX领域/技术中，已见有关...的报道，未见权利要求X所要求的保护的技术方案，
   该技术方案具备新颖性，符合中华人民共和国专利法第二十二条第二款有关新颖性的规定。
```

**关键要点**:
- 必须逐项对比每个对比文件的技术方案
- 明确指出本专利与对比文件的区别技术特征
- 引用专利法第22条第2款作为法律依据

#### F.2 创造性评述

**写作规范**（严格参照参考文档格式）:

```
二、创造性
   在XXX领域/技术中，权利要求X相较于现有技术的区别技术特征为...
   
   该技术方案所解决的技术问题和有益效果在于...
   
   [分析对比文件是否给出技术启示]
   对比文件X公开了...，对比文件未公开区别技术特征，所属技术领域也不相同，
   未给出明确的技术启示，具有突出的实质性特点和显著的进步，
   权利要求X所要求保护的技术方案具备创造性，符合中华人民共和国专利法
   第二十二条第三款有关创造性的规定。
```

**关键要点**:
- 明确列出区别技术特征
- 分析技术问题与有益效果
- 论证对比文件是否给出技术启示
- 引用专利法第22条第3款作为法律依据

#### F.3 实用性评述

**写作规范**:

```
三、实用性
   本专利保护的技术方案可以在产业上制造和使用，因此符合中华人民共和国专利法
   第二十二条第四款的规定。
```

#### F.4 检索结论（五维度综合评价）

**写作规范**（严格参照参考文档格式）:

```
2. 检索结论：

[背景说明 - 如奖项申报或评价目的]
（例如：浙江省人民政府质量奖（省知识产权奖）... 评选方法及流程可参见...）

上述评价指标主要参考专利质量、技术先进性、实施成效、运用及保护措施四个方面，
基本与被广泛认可的法律、技术、经济和前景四个评价要素一一对应。

--- 以下为五维度详细评价 ---

### 技术价值评价
（创新性、技术复杂度、先进性、可替代性、技术成熟度）

### 法律价值评价
（权利要求稳定性、保护范围合理性、被规避风险、文本质量）

### 市场价值评价
（行业概述、市场规模、竞争格局、应用场景、商业化前景）

### 经济价值评价
（成本效益分析、许可转让潜力、投资回报、资产价值）

### 战略价值评价
（产业链地位、技术壁垒、标准必要性、政策契合度、组合价值）

--- 综合评价总结 ---

[总结性评价，包括优势、不足、建议]
```

**五维度评价要点**:

**技术价值**:
1. 创新性：与对比文件的核心技术差异，是否属于实质性突破
2. 技术复杂度：技术壁垒高度、实现难度
3. 先进性：与国际/国内领先水平的对比
4. 可替代性：是否存在替代技术方案
5. 技术成熟度：实验室阶段/中试阶段/产业化阶段

**法律价值**:
1. 权利要求稳定性：独立权利要求保护范围是否合理
2. 从属权利要求层级支撑是否充分
3. 是否存在被无效的风险点
4. 文本质量评估
5. 被规避风险分析

**市场价值**:
1. 行业概述与发展趋势
2. 目标市场规模与增长预测
3. 竞争格局与核心企业分析（含产能、成本、市占率）
4. 应用场景与商业化路径
5. 国家政策与行业标准

**经济价值**:
1. 成本效益分析
2. 技术许可与转让潜力
3. 投资回报与收益预测
4. 专利质押融资、技术入股等可能性

**战略价值**:
1. 产业链定位与技术壁垒
2. 标准必要性与政策契合度
3. 专利组合协同效应
4. 知识产权战略支撑力

## 强制工具使用

1. **`python-docx`**: 解析 `.docx` 格式的专利文件
2. **`mcp-clickhouse_run_query`**: 查询 CNKI 和 WOS 学术文献元数据
   - 数据库: `CERS`
   - 表名: `CNKI` 或 `WOS`
   - **必须使用此工具进行学术文献检索，不可跳过**
3. **`web_search`**: 检索行业信息、竞争企业数据、政策文件
4. **`write_file`**: 输出最终报告文件

## 评分标准

各维度采用 1-10 分制：
- **9-10 分**: 国际领先，具有重大技术突破与市场价值
- **7-8 分**: 国内领先，具有显著创新性与商业化潜力
- **5-6 分**: 国内中等水平，具有改进创新与一定应用价值
- **3-4 分**: 跟随型技术，创新性与市场价值有限
- **1-2 分**: 技术落后或保护范围过窄，价值较低

## 注意事项

1. 封面上的检索报告编号，有请填写，没有请空着
2. "检索使用的分类号"采用 IPC 分类号（国际专利分类）
3. "相关非专利文件"中期刊名称可使用公认的国际惯例缩写
4. "评述和结论"部分的空白处不足时，应当使用 A4 纸作为附页续写
5. 检索结果仅供参考，不作为任何法律依据

## 输出要求

1. 报告必须采用 Markdown 格式，保存到 `/mnt/user-data/outputs/` 目录
2. 文件名格式：`专利检索评价报告_[申请号或专利号].md`
3. 报告必须严格按照 A/B/C/D/E/F 章节结构输出
4. 新颖性/创造性/实用性评述必须引用具体的对比文件编号与内容
5. 技术对比分析必须逐项对比技术方案的关键特征
6. 市场分析必须包含具体的竞争企业名称与数据
7. 学术文献引用必须标注来源（CNKI/WOS）
8. 综合评价必须给出明确的评分与建议
